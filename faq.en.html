Frequently Asked Questions

Q: What is the list of compatibles iOS devices?

The main requirement for Core Secret to work on an iOS device is to have the Bluetooth 4.0 wireless technology. The list of currently compatibles devices is:

  • All iPhone devices since iPhone 4S
  • All iPad devices since iPad 3rd generation, including iPad Air
  • All iPad mini devices
  • iPod touch 5th generation

Airdrop compatibility

Furthermore it is also important to be aware that some of the older iOS devices despite having Bluetooth 4.0 are not compatible with Airdrop (a functionality introduced in iOS 7 and used by Core Secret). In this case the sharing functionality implemented by Core Secret in one of its feature would not be available on these devices. The devices incompatibles with Airdrop are the iPhone 4S and the iPad 3rd generation. Also, compatibles devices need to have activated their iCloud account to use Airdrop.

Q: What iOS versions are supported?

Currently Core Secret requires iOS 7 and is not compatible with previous versions of iOS.

Q: Is the Core Secret's source code available?

Currently only the crypto code. All the crypto code is available as an open source project in this repository.

Q: How to handle Bluetooth 4.0 failures to share or recover secret shares?

  1. Retry the share / recover operation, by returning in the previous menu and then retrying.
  2. If it still doesn't work, try to relaunch the application, try to disable then reenable the Bluetooth in the settings, eventually try to completely restart the device.
  3. In the extreme case if it still doesn't work you can try to reset all the network settings of your device: Settings > General > Reset > Reset Network Settings. Warning! this action will reset all your network settings therefore be prudent only use this method in last resort and only if you know what you're doing.

Q: Core Secret can't handle requests after a device has restarted?

Core Secret uses a special background mode to handle Bluetooth 4.0 requests even when the application is not active. However when the device is restarted applications with Bluetooth 4.0 background services are not automatically restarted, so after a reboot any request sent to this device will fail. This is a current limitation of iOS, to overcome this limitation Core Secret must be manually relaunched by the user after the restart and then its background mode will resume working as before.

Q: How to save and restore secrets and secret shares?

By design it is not valid to make a backup of Core Secret's data and restore it on a different device. See the following links for a rationale and for alternative methods to save secrets and secret shares: Restoring secrets, Secret resharing and Duplicating a secret.

Q: With how many peers is it possible to share a secret?

Currently Core Secret has been tested between 4 peers, the protocol itself could theoretically support 254 peers but the practical limit is likely to be a lot less and depends on the number of Bluetooth 4.0 connections the device is able to handle simultaneously. Maybe around 10 peers but also maybe less depending on the degradation of the signal. Note that this section will be updated once I have more data on the subject.

Q: What data are sent to Core Secret's server, to iCloud's server or to any other internet server?

None. None of Core Secret functionalities need to send some data to any server or use any server. iCloud is not used to save any data whatsoever. Core Secret doesn't send any data to a third party server on the internet, in particular Core Secret doesn't use any analytics services to profile its user nor a library for sending debug informations after a crash. Moreover if needed, Core Secret may even be used without Wifi or without Broadband network enabled, at its core it only needs Bluetooth 4.0.
