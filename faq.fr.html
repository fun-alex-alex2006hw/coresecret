Questions Fréquentes

Q : Quelle est la liste des appareils iOS compatibles?

Pour pouvoir fonctionner avec Core Secret les appareils doivent posséder le Bluetooth 4.0. Les périphériques actuellement compatibles sont :

  • Tous les iPhone à partir du iPhone 4S
  • Tous les iPad à partir du iPad 3ème génération, incluant l'iPad Air
  • Tous les iPad mini
  • Le iPod touch 5ème génération

Compatibilité Airdrop

Par ailleurs certains modéles compatibles Bluetooth 4.0 ne sont pas pourvus de la fonctionnalité Airdrop introduite dans iOS 7. Dans ce cas certaines fonctionnalités de partage de Core Secret utilisant Airdrop seront indisponibles. Les appareils n'ayant pas Airdrop sont le iPad 3ème génération et le iPhone 4S. Pour tous les autres appareils Airdrop est disponible à condition d'avoir activé un compte iCloud.

Q : Quelles versions de iOS sont supportées?

Actuellement Core Secret nécessite l'installation de iOS 7 et n'est pas compatible avec les versions précédentes.

Q: Le code source de Core Secret est-il disponible?

Actuellement, seulement le code crypto. Tout le code crypto est disponible sous la forme du projet open source CryptoPill.

Q : Que faire si le partage ou la reconstitution de secret échoue à cause de la connexion Bluetooth 4.0?

  1. Dans un premier temps réassayez une nouvelle fois de partager/reconstituer le secret en revenant dans le menu précédent puis en recommençant.
  2. Si çà ne fonctionne toujours pas, essayez de relancer l'application, ou essayez de désactiver le Bluetooth puis de le réactiver, ou finalement de redémarrer complétement l'appareil.
  3. Dans le cas extrême où çà ne fonctionnerai toujours pas vous pouvez tenter de réinitialiser les réglages réseau de votre appareil : Réglages > Général > Réinitialiser > Réinitialiser les réglages réseau. Attention! cette manipulation entrainera la perte de tous vos réglages réseau, par conséquent à utiliser seulement en connaissance de cause et avec prudence.

Q : Core Secret n'est pas capable de recevoir de requêtes après avoir redémarré l'appareil?

Core Secret utilise un mode d'exécution spécial pour pouvoir traiter les requêtes Bluetooth 4.0 reçues lorsque l'application est en arrière plan. Toutefois, quand un appareil est redémarré les applications utilisant ce mode ne sont pas automatiquement redémarrées par iOS et donc toutes les requêtes reçues échoueront. C'est une limitation actuelle de iOS, la seule manière de contourner ce problème et de redémarrer manuellement Core Secret après le redémarrage de l'appareil. Ainsi le mode d'exécution en arrière plan sera réactivé.

Q : Comment sauvegarder ou restaurer les secrets?

Il n'est pas possible de restaurer les secrets de manière automatique sur un nouvel appareil. Cette question est traitée dans les sections suivantes de la documentation : Restauration des secrets, Repartage de secret et Dupliquer un chiffré.

Q : Avec combien de pairs est-il possible de partager un secret?

Actuellement l'application a été testée entre 4 pairs, le protocole pourrait en supporter 254 mais la limitation est sans doute bien inférieure et dépend vraissemblablement du nombre de connexions Bluetooth 4.0 simultanées possibles, a priori aux alentours de 10 voir moins si le signal se dégrade. Cette section pourra être mise à jour lorsque j'aurai plus d'informations sur le sujet.

Q : Quelles données sont envoyées sur le serveur de Core Secret, sur iCloud, ou sur un autre serveur sur internet?

Aucunes. Core Secret n'a pas de serveur; n'utilise pas iCloud pour sauvegarder quoi que ce soit et n'envoit aucune donnée sur internet vers un quelconque autre serveur. L'application ne partage les secrets qu'avec les pairs sélectionnés lors des partages de secrets. Si besoin est, il est même possible d'utiliser Core Secret uniquement avec le Bluetooth 4.0 activé, voir cette section pour plus d'informations sur un scénario d'utilisation dérivé de cette idée. Par ailleurs Core Secret n'utilise aucune librairie pour analyser les usages de l'utilisateur, ni aucune librairie pour envoyer des rapports de bugs lorsque l'application crash.
